 <li>What is JSX, and why is it used?</li>
  <p>Ans: Without JSX, writing React components involves a lot of nested function calls that are hard to read; with JSX, you can write your UI structure in a way that’s visually identical to the output.</p>
  <li>What is the difference between State and Props?</li>
  <p>Ans: Props are like arguments to a function. They allow you to pass data from a parent component down to a child. A child component should never modify its own props; it should treat them as a fixed set of instructions. </br> State is like a component’s private "brain." It is used for data that is expected to change based on user interactions</p>

  <li>What is the useState hook, and how does it work?</li>
  <p>Ans: useState is a Hook in React used to create and manage state in a functional component.</p>

  <li>How can you share state between components in React?</li>
  <p>Ans: Sharing state is one of the most common challenges in React. The strategy you choose depends entirely on how far apart the components are in your application tree. </p>

  <li>How is event handling done in React?</li>
  <p>Ans: Event handling in React is very similar to handling events on DOM elements, but there are a few key syntax and architectural differences that make it "the React way."</p>
